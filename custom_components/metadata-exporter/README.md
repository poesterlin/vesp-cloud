# HA Metadata Exporter

Home Assistant integration focused on exporting Home Assistant metadata for editor tooling. Notification helpers are available as an optional add-on.

## Features

- **Metadata export** — WebSocket API that exposes all HA entities, services, devices, and areas for the web editor
- **Custom panel** — built-in dashboard panel at `/esphome-display`
- **Optional notifications** — `input_text` and `input_select` helpers can be auto-created for display overlays

## Architecture

```
HA automation → input_text.set_value("Back door opened")
                        ↓
        input_text.notification_body = "Back door opened"
                        ↓
        ESP32 display reads via bind_ha_string → shows notification
                        ↓
        User dismisses → ESP32 calls input_text.set_value("") → notification cleared
```

No custom services. Any HA automation or script can trigger a notification by setting the helper entities.

## Exporter focus

The primary purpose of this integration is metadata export for the visual editor:

- Entities with safe attribute lists, state metadata, and domain hints
- Services with inferred fields and selector hints
- Devices and areas with relationships used for editor pickers

The integration exposes the websocket command `esphome_display/export`, which the web editor consumes to build entity/service/device selectors.

## Auto-created entities

| Entity | Domain | Purpose |
|--------|--------|---------|
| `notification_title` | `input_text` | Notification header |
| `notification_body` | `input_text` | Notification text |
| `notification_severity` | `input_select` | Severity: `info`, `warning`, `question`, `critical` |

Created on integration setup if they don't already exist. Appear under **Settings → Devices & Services → Helpers**.

## Configuration

### UI (recommended)

**Settings → Devices & Services → Add Integration → HA Metadata Exporter**

| Field | Description |
|-------|-------------|
| Notifications | Enable optional notification helper entities |
| Default Severity | `info` / `warn` / `alert` / `question` |

### YAML

The integration is config-flow based. Add it from UI and adjust settings there.

## Sending notifications

Use **Developer Tools → Services** or any automation:

```yaml
# Set the message
service: input_text.set_value
data:
  entity_id: input_text.notification_body
  value: "Laundry is done"

# Set the title (optional)
service: input_text.set_value
data:
  entity_id: input_text.notification_title
  value: "Washer"

# Set severity (optional, defaults to info)
service: input_select.select_option
data:
  entity_id: input_select.notification_severity
  option: warning
```

All three can be called in one automation action sequence.

## Metadata export

The websocket command `esphome_display/export` returns all HA metadata (entities, services, devices, and areas) with sensitive attributes stripped. This payload is consumed by the web editor to power pickers and codegen configuration.

## Display firmware

The display firmware (generated by the web editor) uses `bind_ha_string` to subscribe to the helper entities. No extra `api.services` configuration is needed — the display reads entity state reactively.

## Installation

### HACS

Add this repository as a custom integration in HACS.

### Manual

```bash
cp -r custom_components/esphome_display /config/custom_components/
```

Restart Home Assistant, then add the integration via UI.

**Requires Home Assistant 2024.1.0+**
